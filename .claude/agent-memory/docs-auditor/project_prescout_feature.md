Prescout data and photo gallery were added as non-config-driven features in March 2026. In April 2026, the shared tables were replaced with per-game tables. In April 2026 (later), photo tagging was added — making photos partially config-driven via `photoTags` and `display.teamView.photoSections`.

**DB tables (per-game, created in `createGame()`, dropped in `deleteGame()`):**
- `prescout_<gameName>` — `team_number UNIQUE`, `data` JSONB; no `game_name` column; isolation by table name
- `photos_<gameName>` — `team_number`, `photo_data TEXT`, `mime_type`, `uploaded_by`, `tag VARCHAR(100)` (nullable), `uploaded_at`; migrated into existing tables via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS tag`

Table names generated by `sanitizePrescoutTableName()` and `sanitizePhotosTableName()` in `schema-generator.js`.

**Photo tagging (added April 2026):**
- `photoTags` is an optional top-level config key: array of `{ name, emoji, color }` defining available tags.
- `GET /api/prescout/photos` accepts optional `?tag=<name>` filter; returns `tag` field in each photo metadata item.
- `POST /api/prescout/photos` accepts optional `tag` in formData.
- Gallery section on `/scout-leads` shows tag pill selector; scouts pick a tag before uploading.
- `display.teamView.photoSections` — array of `{ tag, placement }` on `display.teamView` renders `TaggedPhotoGrid` components inline on `/team-view`. Supported placements: `"aboveEpaChart"`, `"sections.auto.afterImageSelect"`.
- `TaggedPhotoGrid.js` in `src/app/team-view/components/` — renders a scrollable row for one tag; uses `LightboxModal`.
- `LightboxModal.js` in `src/app/components/` — reusable fullscreen lightbox used by both `PhotoGallery` and `TaggedPhotoGrid`.
- `config-validator.js` validates `photoTags` and `display.teamView.photoSections`.

**Page integration (current):**
- `/team-view`: `PrescoutSection` + config-driven `TaggedPhotoGrid` rows (no `PhotoGallery` button)
- `/compare`: `PrescoutSection` only (no photo UI)
- `/scout-leads`: standalone **Gallery section** (team input, drag-and-drop upload, tag selection, photo grid) + `PhotoGallery` button per entry card
- `/admin/prescout`: prescout upload/clear admin UI

**Key behavioral nuances:**
- `photos/[id]` GET and DELETE routes require `?gameId=<id>` query param.
- Photos have no access control beyond "any authenticated user".
- Spreadsheet must have sheet named exactly `"Prescout"` (case-sensitive). Transposed layout.
- `xlsx` (SheetJS) package used for server-side parse.

**Why non-config-driven (prescout):** Prescout data shape varies per event and is imported externally, not collected via the scouting form. Tags are config-driven but optional.

**How to apply:** When reviewing docs, note that prescout/photo features straddle the line — prescout upload is independent of game config, but photo tags and team-view display are config-driven. The old shared-table architecture is gone.
